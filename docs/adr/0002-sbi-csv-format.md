# ADR-0002: SBI証券CSVの2フォーマット対応

## Status

Accepted

## Context

SBI証券のCSVエクスポートには少なくとも2種類のフォーマットが存在することが判明した。

| フォーマット | ヘッダー行の特徴 | 銘柄コードの持ち方 |
|---|---|---|
| A: 保有証券一覧 | `銘柄コード` 列が独立して存在 | 独立した列 |
| B: ポートフォリオ一覧 | `銘柄（コード）` 列に銘柄名も含まれる | `"1414 ショーボンド"` 形式 |

実際に提供されたCSVはフォーマットBであり、当初実装したフォーマットAのパーサーでは `銘柄コード` ヘッダーが見つからずエラーになった。

## Decision

ヘッダー行を動的にスキャンしてフォーマットを自動判別する。

- `銘柄コード` を含む行 → フォーマットA として処理
- `銘柄（コード）` を含む行 → フォーマットB として処理
  - 正規表現 `^(\d{4})\s+(.+)$` で銘柄コードと銘柄名を分離

エンコーディングは CP932 を先に試し、失敗したら `utf-8-sig` → `utf-8` の順でフォールバックする。

## Consequences

- SBIの別画面からエクスポートしたCSVも同じコマンドで読み込める
- 他のフォーマットが今後発見された場合は `_find_header_row` の判別ロジックと対応する `_parse_row_format_*` メソッドを追加する形で拡張できる
