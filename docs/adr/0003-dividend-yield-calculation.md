# ADR-0003: 配当利回りの計算方法

## Status

Accepted

## Context

yfinance が提供する配当利回り関連フィールドはいずれも日本株に対して信頼性が低いことが判明した。

### 問題1: `dividendYield` のフォーマット不一致

`dividendYield` は米国株では小数（0.025 = 2.5%）で返るが、一部の日本株では%表記（3.24 = 3.24%）で返る。そのままPercentage VOに渡すと×100されて324%になる。

### 問題2: `trailingAnnualDividendYield` の株式分割未対応

`trailingAnnualDividendYield` は株式分割の調整が不完全な場合がある。

例: スターゼン（8043）が2025-03-28に3:1分割。`trailingAnnualDividendRate` は分割前の110円をそのまま保持しており、110÷1285 = 8.56%（正しくは約3.3%）になった。

### 問題3: `.dividends` の分割日跨ぎ

`.dividends` 履歴から過去12ヶ月を単純合計する方法でも問題が起きる。

例: 日本製鉄（5401）が2025-09-29に5:1分割。同日付の配当エントリ60円（未調整の疑い）と2026-03-30の12円が合算され、72÷592 = 12.16%になった。

### 観察

`.dividends` の調整状況は銘柄ごとに異なる。

- スターゼン: 分割後実績あり → `.dividends` は調整済み（36.67円 = 110÷3）
- ショーボンド: 分割直後で実績なし → `.dividends` は調整済み（48.4円が12ヶ月合計として正しい）
- 日本製鉄: 分割日当日のエントリが未調整の疑い

## Decision

`.dividends` 履歴から自前で計算する。分割への対応は以下のロジックによる。

```
過去24ヶ月以内に株式分割がある場合:
  分割後の配当実績がある かつ days_since_split >= 180
    → 分割後の配当合計 × (365 / 分割からの経過日数) で年換算
  分割後の配当実績がある かつ days_since_split < 180
    → 分割日当日のエントリを除外した trailing 12ヶ月ウィンドウにフォールバック
      （分割後データが半期1回分未満では年換算誤差が大きいため）
  分割後の配当実績がない（分割直後）
    → .dividends は既に調整済みとみなし、通常の12ヶ月ウィンドウにフォールバック

分割なし:
  → 過去12ヶ月の .dividends 合計をそのまま使用
```

### 問題4: 分割後データ不足による年換算誤差（追記）

伊藤忠商事（8001）が 2025-12-29 に 5:1 分割。分割後97日時点で配当実績が1件（22円）のみ。
22 × 365/97 ≈ 82.8円 → 4.05%（実際の約2倍）となった。

原因: 半期払い銘柄で分割後1支払期間（約180日）未満のデータを年換算すると誤差が大きい。
対応: `days_since_split < 180` のときは分割日当日エントリを除いた trailing 12m にフォールバック。
yfinance は分割前の配当を調整済みで返すため、trailing 12m でも正しい値が得られる。

## Consequences

- 分割日当日のエントリ（日本製鉄の60円）を12ヶ月ウィンドウから排除でき、異常値を防げる
- 分割直後の銘柄（ショーボンド）は `.dividends` の調整済みデータをそのまま使うため正しい値が得られる
- 分割後データが1支払期間未満の銘柄（伊藤忠）は trailing 12m にフォールバックし正しい値が得られる
- `.dividends` の調整状況がyfinanceの挙動変化で変わった場合は再度検証が必要
- yfinance の日本株配当データは根本的に信頼性が低い（公式も非米国株データの精度を保証していない）ため、表示値はあくまで参考値として扱うこと
